## 1. 系统简介

### 1.1 产品概述
**基于空气污染感知的智能温度预测系统**是一款基于深度学习技术的智能温度预测软件。系统采用长短期记忆网络（LSTM）模型，通过分析历史环境参数数据，实现对环境温度的智能预测。

### 1.2 主要特点
- **智能预测**：基于LSTM深度学习模型，准确预测环境温度
- **多方式输入**：支持手动输入和文件导入两种预测方式
- **可视化展示**：直观展示预测结果与实际温度的对比曲线
- **历史追踪**：完整记录每次预测历史，便于分析和回溯
- **结果导出**：支持将预测结果导出为CSV或Excel格式

### 1.3 应用场景
- 空气质量监测站
- 环境监测系统
- 气象预报辅助工具
- 工业生产环境监控
- 科研数据分析

## 3. 安装与配置

### 3.1 安装步骤

#### 步骤1：安装Python
1. 访问 Python 官网（https://www.python.org/）
2. 下载适合您系统的 Python 3.6+ 版本
3. 运行安装程序，**务必勾选**"Add Python to PATH"选项
4. 完成安装

#### 步骤2：安装依赖库
打开命令提示符或PowerShell，执行以下命令：

```bash
pip install numpy pandas torch scikit-learn joblib matplotlib openpyxl
```

或者使用requirements.txt文件：

```bash
pip install -r requirements.txt
```

#### 步骤3：准备训练数据
1. 准备数据集文件 `AirQualityUCI.csv`
2. 数据集应包含以下列：CO(GT)、C6H6(GT)、NO2(GT)、NOx(GT)、RH、AH、T

#### 步骤4：训练模型
1. 确保 `AirQualityUCI.csv` 文件位于程序目录下
2. 运行训练脚本：

```bash
python train.py
```

3. 训练完成后会自动生成以下文件：
   - `model_weights.pt`（模型权重文件）
   - `scaler_X.pkl`（特征标准化器）
   - `scaler_y.pkl`（标签标准化器）
   - `X_test.pt`（测试集特征）
   - `y_test.pt`（测试集标签）

#### 步骤5：启动应用
运行预测界面程序：

```bash
python predict_input.py
```

### 3.2 首次使用配置
1. 首次运行需要先完成模型训练
2. 确保所有必需文件（.pt和.pkl）存在于程序目录
3. 检查系统字体是否支持中文显示

## 4. 系统功能

### 4.1 模型训练模块

#### 4.1.1 功能描述
模型训练模块负责从历史数据中学习温度变化规律，生成可用于预测的神经网络模型。

#### 4.1.2 核心算法
- **神经网络架构**：双向LSTM（长短期记忆网络）
- **隐藏层大小**：128
- **网络层数**：2层
- **Dropout率**：0.2（防止过拟合）
- **优化器**：Adam
- **学习率**：0.001
- **学习率调度**：ReduceLROnPlateau（自适应调整）

#### 4.1.3 数据处理
- **时间窗口**：24个时间步长
- **特征标准化**：StandardScaler标准化
- **数据分割**：训练集70%，验证集15%，测试集15%
- **随机性**：关闭数据打乱（保持时间序列特性）

#### 4.1.4 训练参数
- **训练轮数**：100 epochs
- **损失函数**：均方误差（MSE）
- **模型选择**：基于验证集表现选择最佳模型
- **输出内容**：保存最佳模型权重和标准化器

### 4.2 手动预测功能

#### 4.2.1 功能描述
用户通过界面手动输入当前环境参数，系统实时预测下一时刻的温度值。

#### 4.2.2 输入参数
系统要求输入以下6个环境参数：
- **CO**（一氧化碳浓度，单位：μg/m³）
- **C6H6**（苯浓度，单位：μg/m³）
- **NO2**（二氧化氮浓度，单位：μg/m³）
- **NOx**（氮氧化物浓度，单位：μg/m³）
- **RH**（相对湿度，单位：%）
- **AH**（绝对湿度，单位：g/m³）

#### 4.2.3 预测机制
1. **输入验证**：系统自动验证输入是否为有效数值
2. **序列构建**：自动获取最近24个时间步的历史数据，结合当前输入构建完整序列
3. **特征归一化**：使用训练时保存的标准化器处理数据
4. **模型推理**：将处理后的数据输入LSTM模型
5. **结果反归一化**：将模型输出转换为实际温度值
6. **结果显示**：在界面显示预测温度（摄氏度）

#### 4.2.4 结果记录
- 自动记录预测时间戳
- 保存预测值到历史记录
- 更新可视化图表

### 4.3 文件导入预测功能

#### 4.3.1 功能描述
支持从Excel文件导入环境参数数据，进行批量预测或多点预测。

#### 4.3.2 支持格式
- **Excel 2007+**（.xlsx）
- **Excel 97-2003**（.xls）

#### 4.3.3 文件要求
- 必须包含以下6个列：CO、C6H6、NO2、NOx、RH、AH
- 至少包含10行有效数据（系统需要构建时间序列）
- 数据必须为数值型，无缺失值
- 列名必须与系统要求一致（区分大小写）

#### 4.3.4 预测流程
1. **文件选择**：点击"文件导入"按钮选择Excel文件
2. **数据验证**：系统验证文件格式和数据完整性
3. **数据提取**：提取最近10行数据作为时间序列输入
4. **特征标准化**：使用训练时保存的scaler_X处理数据
5. **模型预测**：输入LSTM模型进行推理
6. **结果反标准化**：使用scaler_y将输出转换为实际温度
7. **结果显示**：显示预测温度并添加到历史记录

### 4.4 可视化展示功能

#### 4.4.1 功能描述
实时显示预测温度与实际温度的对比曲线，支持交互式操作。

#### 4.4.2 显示内容
- **预测温度曲线**：蓝色虚线，显示所有历史预测值
- **实际温度曲线**：红色实线，显示已有的实际温度数据
- **最新预测点**：红色圆点标记，突出显示最新预测值
- **网格线**：半透明网格，便于读取数值
- **图例**：清晰标注各线条含义

#### 4.4.3 数据处理
- **数据平滑**：使用5点均匀滑动窗口平滑曲线，减少波动
- **时间轴**：自动生成时间戳，格式为"月-日 时:分"
- **坐标自适应**：自动调整Y轴范围以最佳展示数据

#### 4.4.4 交互功能
- **缩放**：鼠标滚轮缩放图表
- **平移**：拖拽移动查看范围
- **重置**：返回初始视图
- **配置**：调整图表参数
- **保存**：将图表保存为PNG格式

### 4.5 结果保存功能

#### 4.5.1 功能描述
将预测历史记录导出为文件，便于后续分析和存档。

#### 4.5.2 保存格式
- **CSV格式**（.csv）：通用数据格式，可用Excel打开
- **Excel格式**（.xlsx）：Microsoft Excel专用格式

#### 4.5.3 保存内容
每次保存包含以下信息：
- **时间**：预测时间戳
- **预测值**：预测的温度值（°C）
- **实际值**：实际温度值（如有，°C）
- **预测类型**：历史数据/手动预测/文件导入

#### 4.5.4 操作步骤
1. 点击"保存结果"按钮
2. 选择保存位置和文件名
3. 选择文件格式（.csv或.xlsx）
4. 点击保存
5. 系统提示保存成功

### 4.6 历史记录管理

#### 4.6.1 记录类型
系统自动维护以下三类预测记录：
- **历史数据**：从测试集加载的初始预测记录
- **手动预测**：通过界面手动输入产生的预测
- **文件导入**：从Excel文件导入产生的预测

#### 4.6.2 记录结构
每条记录包含：
- 时间戳（字符串格式）
- 预测值（浮点数）
- 实际值（浮点数，可为空）
- 类型标识（字符串）

#### 4.6.3 记录限制
- 历史记录在程序运行期间保存于内存
- 程序关闭后需要重新加载
- 建议定期使用"保存结果"功能备份数据

## 6. 常见问题

### 6.1 安装相关问题

#### Q1: 提示"找不到tkinter模块"
**A:** tkinter是Python自带的GUI库。Windows用户通常会自动安装。如遇问题，尝试重新安装Python并确保勾选"tcl/tk and IDLE"选项。

#### Q2: 提示"无法导入torch"
**A:** 运行 `pip install torch` 安装PyTorch。如果仍然报错，访问PyTorch官网获取针对您系统的安装命令。

#### Q3: 中文显示为乱码
**A:** 系统使用SimHei（黑体）字体。确保您的系统安装了中文字体库。Windows系统通常自带该字体。

#### Q4: Excel文件无法打开
**A:** 需要安装openpyxl库：`pip install openpyxl`

### 6.2 训练相关问题

#### Q5: 训练数据文件找不到
**A:** 确保AirQualityUCI.csv文件位于与train.py相同的目录下。检查文件路径和文件名是否正确。

#### Q6: 训练过程出错
**A:** 检查数据集格式是否正确，确保包含必需的列，且数据无异常值。查看控制台错误信息定位具体问题。

#### Q7: 训练时间过长
**A:** 训练100轮可能需要10-30分钟，取决于您的计算机性能。可以修改train.py中的epochs数量减少训练时间（用于测试）。

### 6.3 预测相关问题

#### Q8: 手动预测提示"尚无历史预测"
**A:** 首次预测需要系统加载历史数据。确保已运行完整训练流程，X_test.pt文件存在于目录中。

#### Q9: 输入参数验证失败
**A:** 
- 检查输入是否为空
- 确保输入为数值（不能包含非数字字符）
- 确保输入为有限数值（不能为inf或NaN）
- 检查小数点的使用（支持小数点）

#### Q10: 文件导入失败
**A:**
- 确保Excel文件包含必需的所有列（CO、C6H6、NO2、NOx、RH、AH）
- 确保列名完全匹配（区分大小写）
- 至少需要10行数据
- 所有数据必须为数值型，无缺失值
- 检查文件是否被其他程序占用

#### Q11: 预测结果明显不准确
**A:**
- 输入的环境参数应在合理范围内
- 检查数据单位是否正确
- 重新训练模型（如果训练数据有更新）
- 预测结果仅供参考，实际应用需结合专业知识判断

### 6.4 界面使用问题

#### Q12: 图表不更新
**A:** 
- 刷新界面或重启程序
- 检查是否有错误提示
- 确保预测操作成功完成

#### Q13: 图表显示空白
**A:**
- 确保已有预测记录
- 重启程序重新加载数据
- 检查数据文件是否完整

#### Q14: 无法保存结果
**A:**
- 确保有预测记录可以保存
- 检查保存路径是否有写入权限
- 确保磁盘空间充足

### 6.5 性能相关问题

#### Q15: 程序启动慢
**A:** 首次启动需要加载模型和预处理数据，需要5-10秒。后续操作实时响应。

#### Q16: 内存占用高
**A:** 系统需要加载模型和历史数据，预计占用200-500MB内存，属正常范围。

## 8. 注意事项

### 8.1 数据质量要求
- 输入的环境参数必须为实际测量的有效数值
- 避免输入异常值或明显错误的数据
- 确保数据的单位和范围符合系统要求

### 8.2 模型使用建议
- 模型基于特定数据集训练，预测效果可能与训练数据分布相关
- 建议定期用最新数据重新训练模型
- 预测结果仅供参考，实际应用需结合专业判断

### 8.3 文件管理
- 定期备份生成的模型文件（.pt和.pkl文件）
- 保存重要的预测历史记录
- 避免修改训练好的模型权重文件

### 8.4 系统维护
- 定期检查依赖库版本，保持与训练时一致
- 避免在同一目录下重复训练导致文件覆盖
- 注意磁盘空间，及时清理不需要的历史文件

### 8.5 使用限制
- 系统仅供学习和研究使用
- 生产环境使用请进行充分的测试和验证
- 不得用于违法或不当用途

### 8.6 技术支持
如遇到技术问题，请检查：
1. Python和依赖库版本是否正确
2. 所有必需文件是否存在
3. 输入数据格式是否符合要求

## 附录：数据格式说明

### 训练数据格式要求

#### CSV格式
```
CO(GT);C6H6(GT);NO2(GT);NOx(GT);RH;AH;T
1.50;5.00;80.00;120.00;45.00;0.50;18.50
...
```

#### 列说明
- **CO(GT)**: 一氧化碳浓度（μg/m³）
- **C6H6(GT)**: 苯浓度（μg/m³）
- **NO2(GT)**: 二氧化氮浓度（μg/m³）
- **NOx(GT)**: 氮氧化物浓度（μg/m³）
- **RH**: 相对湿度（%）
- **AH**: 绝对湿度（g/m³）
- **T**: 温度（°C）

### 预测输入文件格式

#### Excel格式（手动导入）
| CO | C6H6 | NO2 | NOx | RH | AH |
|----|------|-----|-----|----|-----|
| 1.5 | 5.0 | 80 | 120 | 45 | 0.5 |

**要求**：
- 必须包含所有6列
- 列名与示例完全一致
- 至少10行有效数据
- 所有数据为数值型
- 无缺失值
